##**Typescript** এর তিনটি কনফিউজিং টাইপ নিয়ে ক্লিয়ার একটা ধারণার জন্য এই ব্লগটি বেশ উপকারী হবে বলে আশা করছি।
বিশেষ ভূমিকায় না যেয়ে সরাসরি চলে যাই টপিকে।
১. any – সহজ ভাষায় এটা অনেকটা "যেমন খুশি তেমন সাজো” ধরণের স্বাধীনতা। 
anyটাইপ  ব্যবহার করলে যা হয় তা হলো, আমাদের Typescript বন্ধ হয়ে যাবে। সে আর কোন টাইপই চেক করবে না। যা খুশি তাই হতে দিবে। 
অতি স্বাধীনতায় স্বকীয়তা হারানোর ভয়ও আছে এখানে। এটা সেফ নয়। শুধু মাত্র অতি জরুরি প্রয়োজন ছাড়া কাউকে এত স্বাধীনতা দেওয়া যায় না। সুতরাং খুব জরুরি ক্ষেত্রেই কেবল আমরা any ব্যবহার করবো।
এই প্রয়োজনীয় ক্ষেত্রগুলাকে ডিফাইন করতে পারি এভাবে- 
পুরোনো JavaScript কোডকে ধীরে ধীরে TypeScript-এ নিয়ে আসার সময়
তৃতীয় পক্ষের লাইব্রেরি যেটার টাইপ নেই
খুব তাড়াতাড়ি প্রোটোটাইপ করার সময় (পরে ঠিক করে নেবো ভাবলে)

একটা এক্সাম্পল দেখি আমরা- 
```ts
let data: any = "Hello";
data = 123;
data.foo(); // কোনো কম্পাইল এরর নেই
```


২. Unknown – এ হচ্ছে "ধরি মাছ না ছুঁই পানি” বান্দা। সে একদিকে দরজা খুলে দিবে যা খুশি তাই ভ্যালু নিতে, আরেকদিকে চেক না করে এক্সিকিউটও করতে দিবে না। 
কোন কোন ক্ষেত্রে এর ব্যবহার হতে পারে সেটা হলো-
API থেকে ডেটা আসা আসলে (JSON.parse, fetch response ইত্যাদি)
ডায়নামিক ডেটা যার টাইপ আগে থেকে জানা নেই
any-এর বদলে সেইফ অপশন চাইলে
এজ ফর এক্সাম্পল- 
```ts
let value: unknown = "Bangla";

// value.toUpperCase(); // এরর
if (typeof value === "string") {
  console.log(value.toUpperCase()); // ঠিক আছে
}
```

৩. never– "শাট আপ” টাইপের ধামকি দিয়ে বসায় রাখবে। কোন কথাই বলতে দিবে না। কোন ভ্যালু কে ভ্যালুই দিতে চায় না। 
এই টাইপ টা ব্যবহার করতে হয় এমন জায়গায় যেখানে আসলে  কোন ভ্যালুই আমাদের দরকার নাই।
যেমন- 
ফাংশন যা কখনো return করে না
exhaustive switch/case চেক এর ক্ষেত্রে 
যখন কোনো ফাংশন কখনো রিটার্ন করে না (থ্রো করে বা ইনফিনিট লুপ), তখন তার রিটার্ন টাইপ never হয়।
ফর এক্সাম্পল- 
```ts
function throwError(msg: string): never {
  throw new Error(msg);
}

function infiniteLoop(): never {
  while (true) {}
}

// Exhaustive check
type Animal = "cat" | "dog";
function checkAnimal(a: Animal) {
  switch(a) {
    case "cat": break;
    case "dog": break;
    default:
      const x: never = a; // TypeScript error ধরবে যদি কোনো কেস বাদ পড়ে
  }
}
```

##Union এবং Intersection টাইপস;
১. Union- স্কুলের ম্যাথ বইএর সেট  ফাংশনে যেমন ছিল, তেমনই একটা বিষয় । অনেক গুলা লিস্টের যেকোন একটা আইটেম মিললেই সেটা পিক করে নিয়ে নেবো। যেমন, আই নিড আ ড্রিংক। তাতে লেবু থাকতে পারে, চিনি থাকতে পারে, লবণ থাকতে পারে, বরফ থাকতে পারে।   আবার এসব কিছু না থেকে শুধু পানিও থাকতে পারে। এই ভার্সাটাইল সিলেকশনের অপশন টাই ইউনিয়ন। 
এটা ‘ | ‘ এই সিম্বল দিয়ে প্রকাশ করা হয়। অর লজিকের এপ্লিকেশন। 
```ts
let value: string | number;
value = "Hello"; // ঠিক আছে
value = 123;     // ঠিক আছে
// value = true; // Error, কারণ boolean এখানে নেই
```

২. Intersection- এটাও ছোটবেলার ম্যাথের থিওরি। এটা হচ্ছে এন্ড লজিকের এপ্লিকেশন। প্রকাশ করা হয় ‘ &' এই সিম্বল দিয়ে।
এখানে আগের ড্রিংক এর মতো যেকোন ড্রিংক হলেই চলবে  না। নির্দিষ্ট সবগুলো উপাদান মিললেই কেবল এটা এক্সেপ্টেড হবে। সব শর্ত না মিললে ইন্টারসেকশন কাজ করে না।
```ts
type Person = { name: string; age: number };
type Employee = { employeeId: number };

type Staff = Person & Employee;

const staffMember: Staff = {
  name: "Hossain",
  age: 25,
  employeeId: 101
};
```
